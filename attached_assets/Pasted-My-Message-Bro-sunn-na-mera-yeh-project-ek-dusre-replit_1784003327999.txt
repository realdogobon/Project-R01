Root Cause Analysis
There is one primary root cause that creates the illusion of many separate bugs. Everything else is downstream of it.

The Real Problem: initialSettings is a Module-Level Constant
DasKeyboardApp.tsx, line 88:

const initialSettings = loadSettings();   ← runs ONCE when JS module first loads
JavaScript modules are loaded and cached once per page load. This line runs a single time — when you first open the app. From that point on, initialSettings is frozen.

The useState initializers on lines 96–104 all read from this frozen object:

useState(initialSettings.rgbEnabled ?? false)
useState(initialSettings.rgbEffect ?? "static")
... etc
What goes wrong when you switch keyboards:

App loads → initialSettings = { rgbEnabled: false, rgbEffect: "static", ... } (defaults, since nothing saved yet)
You turn RGB on, set wave effect, adjust brightness → these get written to localStorage correctly ✓
You switch to Classic → DasKeyboardApp unmounts completely
You switch back to Das → DasKeyboardApp remounts from scratch, useState initializers run again
They read from initialSettings — which is still the frozen object from step 1, NOT from localStorage
Result: RGB is off, effect is static, brightness is wrong — all your changes are gone
The localStorage save is working. The restore is broken.

The Fix Plan (2 surgical changes, same file)
Change 1 — Remove the module-level constant, use lazy initializers:

Instead of const initialSettings = loadSettings() at module scope, call loadSettings() inside each useState initializer (lazy function form). React only calls lazy initializers on the first render of each mount, so this reads fresh from localStorage on every mount — exactly what's needed.

Delete line 88 entirely. Change all 5 affected useState calls to lazy form. This is the complete fix for all persistence bugs.
